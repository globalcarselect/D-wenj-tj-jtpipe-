# 部署步骤说明

## 当前问题
`wordpress.ts` 文件缺少 `export` 语句，导致模块导入失败。

## 解决方案

### 步骤1：复制修复后的文件到服务器

在您的本地机器（Windows）上，运行以下命令：

```powershell
# 复制修复后的wordpress.ts文件到服务器
Copy-Item "d:\wenj\独立站\tj-jtpipe\DM\src\services\wordpress.ts" "C:\Websites\jtpipeline\src\services\wordpress.ts" -Force
```

### 步骤2：验证文件已正确复制

```powershell
# 验证文件存在
Test-Path "C:\Websites\jtpipeline\src\services\wordpress.ts"

# 查看文件最后几行，确认有export语句
Get-Content "C:\Websites\jtpipeline\src\services\wordpress.ts" | Select-Object -Last 10
```

### 步骤3：重新构建项目

```powershell
# 切换到项目目录
cd C:\Websites\jtpipeline

# 清理之前的构建
npm run clean

# 重新构建
npm run build
```

### 预期结果

构建应该成功，输出类似：

```

✓ Compiled successfully
✓ Linting and checking validity of types  
✓ Collecting page data  
✓ Generating static pages (X/X)
✓ Finalizing page optimization  
```
